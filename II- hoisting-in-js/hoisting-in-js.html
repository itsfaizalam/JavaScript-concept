JavaScript Hoisting Notes

Defination: Hoisting JavaScript ka ek behavior hai jismein variable aur function declarations ko unke scope ke shuruat mein upar le jaya jata hai. Matlab, aap variables aur functions ko unke declare hone se pehle bhi use kar sakte hain, lekin unki values undefined hoti hain jab tak unhe initialize nahi kiya jata.

Example 1: Basic Variable Hoisting


console.log(x); // Output: undefined
var x = 5;
console.log(x); // Output: 5

Pehla console.log(x); call undefined print karta hai kyunki var x; hoisting ki wajah se upar aa jata hai, lekin uska initialization 5 nahi hua hai. Dusra console.log(x); par 5 print hota hai kyunki ab x ko value mil chuki hai.

Example 2: Function Hoisting


greet(); // Output: "Hello!"
function greet() {
    console.log("Hello!");
}

Function greet() ko call karne se pehle hi hum isse use kar rahe hain, aur ye sahi tarike se kaam karta hai kyunki function declaration hoisting ke through upar aa gaya hai.

Example 3: Let and Const Hoisting


console.log(a); // Output: ReferenceError: Cannot access 'a' before initialization
let a = 10;

console.log(b); // Output: ReferenceError: Cannot access 'b' before initialization
const b = 20;

let aur const variables hoisted hote hain, lekin unhe temporal dead zone (TDZ) ke wajah se unke declare hone se pehle access nahi kiya ja sakta. Isliye pehle console.log(a); aur console.log(b); par ReferenceError aata hai.

Example 4: Function Expression Hoisting


console.log(c); // Output: ReferenceError: Cannot access 'c' before initialization
var c = function() {
    console.log("I am a function expression!");
};

c(); // Output: "I am a function expression!"

Yahan c ek function expression hai. Pehle console.log(c); par ReferenceError aata hai kyunki function expression hoisting nahi hoti. Lekin jab hum c(); call karte hain, tab function properly call hota hai kyunki ab c ko value mil chuki hai.
